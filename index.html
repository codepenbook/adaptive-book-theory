Описание

Адаптивная вёрстка сайта позволяет веб-страницам автоматически подстраиваться под экраны планшетов и смартфонов. Мобильный интернет-трафик растёт с каждым годом и чтобы эффективно обрабатывать этот трафик, нужно предлагать пользователям адаптивные сайты с удобным интерфейсом.

Введение

И так в целом можно выделить 4 подхода к верстке, это:

- Фиксированная верстка (не адаптивная) - Отзывчивая верстка (резиновая) - Адаптивная верстка (с брейкпоинтами) - Отзывчивая-адаптивная верстка (резиновая с брейкпоинтами)

Отзывчивая верстка (резиновая)

Здесь мы записываем значения в процентах для наших блоков, после чего они как бы становятся резиновыми.

Чтобы добиться такого эффекта в head > meta указано name="viewport" и content="width=device-width", а в style.css мы снимаем ограничения минимальной ширины для body и выставляем max-width: 1200px для container.

Так для первых трёх блоков напишем значение flex: 0 1 33.333% и теперь они будут занимать всю доступную ширишу, выстроившись в три "колонки".

Аналогично и с блоками ниже, но уже в свойстве flex необходимо записать flex: 0 1 50%, т.к. у нас по сути два блока, первый большой и второй, в котором находятся еще два блока.

Адаптивная верстка (с брейкпоинтами)

При адаптивной верстке используются брейкпоинты, в которых мы укажем изменения max-width в container. Делается это для того, чтобы сайт безопасно отображался при различных разрешениях экрана устройства.


Такие брейкпоинты имееют запись:

@media (max-width: 1200px) {

.container {

max-width: 970px;

}

}


@media (max-width: 992px) {

.container {

max-width: 750px;

}

}


@media (max-width: 767px) {

.container {

max-width: none;

}

}

Отзывчивая-адаптивная верстка (резиновая с брейкпоинтами)

Такая верстка совмещает в себе отзывчивость и адаптивность.


Тут нам уже не надо записывать брейкпоинты, в которых мы указывали max-width в container ранее.


Главная наша задача состоит в том, чтобы совместить отзывчивость и адаптивность.


И проблему, которую мы решим это то, что когда мы использовали отзывчивую(резиновую) верству и уменьшали размер экрана, то блоки становились все уже и уже, если бы в них был какой-то контент, то на ширине ниже 767px, врядли мы смогли его адекватно прочитать.



Поэтому для ширины экрана 767px мы зададим изменение раположения наших блоков. Так для первых трёх блоков запишем следующее:

@media (max-width: 767px) {

.one-list {

/* Свойство column выстроит наши блоки не вряд, а в колонку */

flex-direction: column;

}

.one-list>*:not(:last-child) {

/* Зададим отступы снизу для каждого блока, кроме последнего */

margin: 0px 0px 20px 0px;

}

.one-column {

/* Так как наши блоки теперь не ряд, а колонкой, то мы можем указать для них 100%, чтобы занимали всю доступную ширишу */

flex: 0 1 100%;

}

}


Аналогично запишим и для блоков ниже:

@media (max-width: 767px) {

.two-body {

flex-direction: column;

}

.two-body>*:not(:last-child) {

margin: 0px 0px 20px 0px;

}

.two-big,

.two-smalls {

flex: 0 1 100%;

}

}
